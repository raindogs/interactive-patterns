## コンセプト

「空気の流れを視覚化するモビル」

吊り下げられた抽象パーツが、見えない風や観客の操作に反応して連鎖的に揺れ、
静と動のバランスが継続的に変化するキネティックアート習作を目指す。

## モチーフ

- アレクサンダー・カルダーのモビルに見られる、軽やかな吊り構造と非対称バランス
- 微風で位相がずれていく複数レイヤーの回転・揺動
- 影が重なって構成を変える「実体 + 投影」の二重表現

## 使用技術（ライブラリ）

- 言語: HTML / CSS / JavaScript
- 描画: Canvas 2D（初期）、必要に応じて WebGL へ拡張
- 主要ライブラリ: なし（初期はVanilla）
- 外部依存を採用する理由: 力学挙動の精度が不足する場合のみ、軽量物理ライブラリ導入を検討

## 適用・応用するシーン

- Web上のジェネラティブアート展示
- インスタレーションの事前プロトタイピング
- ポートフォリオ向けの動的ビジュアル研究

## 体験ゴール

- 鑑賞者の操作が即時ではなく慣性を伴って伝搬し、余韻のある動きを感じられる
- 画面を放置しても、微小なゆらぎで構図が変化し続ける
- パーツ同士の距離・速度の変化が音のないリズムとして知覚される

## インタラクション仕様

- 開始条件: 初回 pointer 操作でシミュレーション開始
- 追従ロジック: 入力ベクトルを上位アームへ加算し、子パーツへ減衰伝搬
- エフェクト発火条件: 角速度閾値を超えたパーツにハイライトまたは軌跡表示
- 消失条件: 軌跡は一定時間でフェードアウト
- 制限値（同時描画数、寿命、密度など）: 初期はパーツ 12-24 個、60fps 維持を優先

## ビジュアル仕様

- 色設計（主要カラー / 参考作品）: オフホワイト背景 + 原色アクセント（赤/青/黄）
- レイヤー構成: 背景 / モビル本体 / 影 / 軌跡エフェクト
- 合成モード・ブラー・にじみ設定: 影のみ軽いブラー、本体はエッジを保つ
- 背景処理: ごく薄いグラデーションで空間の奥行きを補助

## パフォーマンス目標

- 目標 FPS: Desktop 60fps / Mobile 50fps
- 計測条件（端末 / ブラウザ）: macOS + Chrome, iOS Safari, Android Chrome
- 劣化戦略（低スペック時のフォールバック）: パーツ数削減、影解像度低下、軌跡無効化

## アクセシビリティ

- `prefers-reduced-motion` 対応: 有効時は揺れ振幅と自動ゆらぎを縮小
- タッチ操作時の代替挙動: ドラッグ優先、連続入力のノイズ抑制
- コントラスト / 可読性の配慮: UI表示時は最低コントラスト比を確保

## 受け入れ条件

- [ ] ポインタ入力でモビル全体に遅延伝搬する揺れが発生する
- [ ] 無操作時でも微小な変化が継続し、静止しきらない
- [ ] Desktop/Mobile 両方で破綻なく操作できる

## テスト観点

- 対象端末: MacBook / iPhone / Android
- 確認手順: パーツ数を変えながら入力レスポンスとFPSを計測
